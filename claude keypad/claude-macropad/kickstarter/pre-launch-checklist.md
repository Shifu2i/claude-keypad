# Claude Macropad — Kickstarter Pre-Launch Checklist

> Kickstarter will not approve a hardware campaign without a working physical prototype.
> All items in the "Hard Requirements" section must be complete before submitting for review.
> Kickstarter's review process typically takes 1–3 business days.

---

## Hard Requirements (Kickstarter Will Reject Without These)

- [ ] Working physical prototype exists and has been assembled
- [ ] Prototype performs all features demonstrated in the campaign (all keys register, both encoders work, USB enumerates)
- [ ] Campaign video filmed showing the **physical prototype** functioning — not simulated, not rendered
- [ ] Main campaign image is a real photograph of the prototype (1024 × 576 px minimum, 16:9)
- [ ] Risks & Challenges field completed (specific, not vague)
- [ ] At least one reward tier created
- [ ] Estimated delivery date set per tier
- [ ] Funding goal calculated and set (covers real minimum cost to manufacture and ship)
- [ ] Creator account identity-verified (Kickstarter ID verification badge)
- [ ] AI disclosure added if any AI-generated images, copy, or renderings are used in the campaign
- [ ] Anthropic trademark usage reviewed — no official Anthropic logos used; fan-project disclaimer visible

---

## Campaign Content

- [ ] `campaign-story.md` — all `[RENDER: ...]` placeholders replaced with real prototype photos or clearly-labelled renders
- [ ] Project title confirmed: *Claude Macropad — A Mechanical Macro Pad for Claude Code*
- [ ] Subtitle / blurb written and under 135 characters
- [ ] Category set: Technology → Hardware
- [ ] Campaign duration set to 30 days
- [ ] All reward tiers built in the Kickstarter editor (from `reward-tiers.md`)
- [ ] Shipping costs set per region per tier (get fulfilment partner quotes first)
- [ ] Add-ons configured in BackerKit (set up BackerKit project before campaign goes live)
- [ ] Stretch goals written and added to campaign story (optional but recommended)
- [ ] FAQ section populated

---

## Renders / Photography

- [ ] Hero image: 3/4 angle assembled unit — real photo, Claude orange case visible
- [ ] Feature callout image: flat lay with component labels
- [ ] Close-up strip: hotswap socket, encoder knob, USB-C port
- [ ] PCB photo: top-down bare board with component layout
- [ ] Case detail: top-down showing robot-face silhouette cutout
- [ ] VIA screenshot or GIF: browser app with Claude Macropad layout loaded
- [ ] Reward tier flat-lay: one image per tier showing box contents
- [ ] Team / creator photo: real headshot or workshop photo
- [ ] Development journey: sketch or early prototype photo alongside final version

> Kickstarter rule: photorealistic 3D renders cannot be used as the **primary** product image. They can appear in the campaign body if clearly labelled "concept render" or "CAD render". Real prototype photos are required.

---

## Video

- [ ] Script written (target: 90–120 seconds)
- [ ] Footage filmed: product demo, key press, encoder rotation, VIA mapping, in-context desk shot
- [ ] Voiceover or on-camera presenter recorded
- [ ] Video edited: 1080p, under 5 GB file size
- [ ] Captions/subtitles added (improves accessibility and autoplay-without-sound viewing)
- [ ] Video uploaded to Kickstarter project editor

**Script structure:**
1. Hook — problem (0–15 s): *"You reach for the same five Claude Code actions hundreds of times a day..."*
2. Product reveal (15–30 s): show the physical unit on desk
3. Demo: plug in, open VIA, remap a key, press it (30–75 s)
4. Key features rapid-fire (75–100 s)
5. Who made it and why (100–110 s)
6. Call to action — back the project (110–120 s)

## Creator Profile

- [ ] Profile photo uploaded (real photo — not a logo)
- [ ] Bio written (2–4 sentences: keyboard background, engineering experience, GitHub link)
- [ ] Website URL set (project site or GitHub repo)
- [ ] Twitter/X, Reddit, GitHub links added
- [ ] Kickstarter identity verification completed (displays trust badge)
- [ ] Have backed at least one other Kickstarter project prior to launching

---

## Pre-Launch Page

- [ ] Pre-launch page enabled in Kickstarter project editor
- [ ] Pre-launch image and teaser text set
- [ ] Pre-launch page URL shared with: r/MechanicalKeyboards, r/MechKeys, r/ClaudeAI, GeekHack, personal Twitter/X, Discord
- [ ] Email list set up (Mailchimp or Klaviyo) — collect signups independently of Kickstarter
- [ ] Target: 500+ pre-launch followers / email subscribers before going live
- [ ] Pre-launch page live at least 4 weeks before campaign start date

---

## Budget & Finance

- [ ] COGS calculated (PCB + PCBA + case + components + packaging + fulfilment handling)
- [ ] Shipping costs confirmed with fulfilment partner per region
- [ ] Kickstarter fees (5%) and payment processing (3% + $0.20 or 5% + $0.05) budgeted
- [ ] Tax advice sought — Kickstarter funds are taxable income in most jurisdictions
- [ ] Funding goal confirmed: covers tooling + minimum production run + fees + 15% contingency
- [ ] Early bird price confirmed (≥20% below standard, still covers COGS + fees)
- [ ] BackerKit account created and project set up

---

## Manufacturing & Legal

- [ ] JLCPCB production-volume quote obtained (100 / 250 / 500 unit tiers)
- [ ] Case production partner identified and quoted
- [ ] Component pre-order quantities confirmed (Pro Micro, EC11, sockets, keycaps)
- [ ] Fulfilment partner contracted (or self-fulfilment plan confirmed for small volumes)
- [ ] Anthropic trademark review self-completed — no Anthropic logos; fan-project disclaimer in place
- [ ] Open-source licence files confirmed: firmware GPL-2.0, PCB CERN OHL-W 2.0, case CC BY-NC-SA 4.0

---

## Community & Marketing

- [ ] r/MechanicalKeyboards post drafted (pre-launch interest check — "interest check" posts are accepted there)
- [ ] GeekHack interest check thread drafted
- [ ] Discord server or channel created for backers
- [ ] Launch-day outreach list: keyboard YouTubers, blogs, community Discord servers
- [ ] Day 1 push plan: personal network, community posts, email list blast timed to campaign go-live
- [ ] Campaign update cadence planned: at minimum monthly; more frequently during critical milestones

---

## Launch Day

- [ ] Campaign submitted for Kickstarter review (allow 1–3 business days)
- [ ] Launch date and time set (Tuesday–Thursday launch, 8–10am in your timezone, performs best)
- [ ] All pre-launch follower notifications will fire automatically at launch — confirm this is enabled
- [ ] Email list blast scheduled for campaign launch time
- [ ] Social posts scheduled / ready to publish at launch
- [ ] Community posts (Reddit, GeekHack) ready to go live at launch
- [ ] Refreshing the campaign page obsessively for the first 48 hours is normal — enjoy the ride
